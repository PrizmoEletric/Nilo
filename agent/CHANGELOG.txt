NILO — CHANGELOG
================
Newest first. Dates are approximate session dates.

────────────────────────────────────────────────────────
2026-05-04
────────────────────────────────────────────────────────
New modules:
- blockresolver.js — safe block resolution via bot.registry (runtime, not hardcoded IDs).
  resolveBlock, getStateIdAt, findNearestBlock, searchRegistry. Works for vanilla and modded.
  Also copied to agent/ as a reference document for future AI sessions.
- easyauth.js — auto-register / auto-login for EasyAuth servers (reads NILO_PASSWORD env var)

Multi-server support (config.js + servers.json):
- servers.json — named server profiles (host, port, version, auth, log_path, description)
- config.js: getServerConfig(), getActiveServerName(), setActiveServer(name), addServer(), loadServers()
- nilo.js: reads from getServerConfig() instead of hardcoded constants
- CLI flag: node nilo.js --server=<name> (or --server <name>) to pick a profile at launch
- In-game: "list servers", "current server", "switch server <name>", "save server as <name>"

"this" command stateId fix (commands/misc.js):
- Block response now includes stateId: "oak_log at X Y Z [sid:N]"
- Entity response includes hostile flag + health: "zombie (hostile mob [hostile] hp:18)"
- Both looked up against DB entities table for hostile classification

Block physics teaching (commands/registry.js + registry-patch.js):
- "X is solid" / "X is passable" / "X is walkable" — teaches Nilo block physics conversationally
- Writes to DB blocks table; registry-patch reads it back to fix pathfinding for modded blocks
- setManualBlockPhysics exported from registry-patch.js
- Physics resolution order: DB → BLOCK_PHYSICS map → name-based heuristic

chatLong (letta.js):
- New export chatLong(bot, text) — splits long Letta replies and sends them as multiple chat messages

Fabric handshake handler (nilo.js):
- Handles login_plugin_request packets: owo:handshake, forgeconfigapiport, and unknown channels
- Prevents Fabric servers from kicking Nilo during the mod negotiation phase
